Печать ценников (версия 2.16)

 

Перед началом печати необходимо сделать несколько предварительных действий:

 

  1. Убедится что правильно выбраны Организация, Тип цен или Склад (если работаем с ценами АТТ). По-умолчанию, эти параметры заполняются из настроек пользователя.
  2. Выбрать тип печатной формы. Формы загружаются из каталога, указанного в параметре "Путь к ценникам" на закладке Настройки. В общем случае, может быть несколько форм. После выбора печатной формы автоматически проставляются настройки формы – количество в строке (количество ценников в строке), количество строк на странице, тип бумаги (портрет или ландшафт), масштаб (0 - по ширене страницы). Данные настройки хранятся в печатной форме и не требуют корректировки.
  3. Тип округления устанавливается по умолчанию до копеек. Этот параметр показывает, как округлять цены в ценниках. Его можно не менять.
  4. Формат цены - как печатать цену, с копейками или без.
  5. Галочка «Включать наименование родителя в наименование товара» означает будет ли включатся наименование группы в наименование товара. Если галочка установлена, то для товара «БЛАЙЗЕР ALAIN MANOUKIAN  P L», находящегося в группе «Одежда женская» наименование будет печататься «Одежда женская БЛАЙЗЕР ALAIN MANOUKIAN  P L». Разделитель - любой произвольный набор символов, разделяющий наименование родительской группы и наименование товара. 
  6. Если в организации учет ведется по характеристикам, необходимо установить параметр "Использовать характеристики товаров".
  7. Необходимо заполнить список товаров, на которые будем печатать ценники. Подбор товаров осуществляется несколькими способами:

·  добавление одной позиции в список из справочника номенклатуры;

·  добавление нескольких позиций;

·  добавление всех товаров из группы номенклатуры;

·  добавление товаров из документа;

·  добавление позиции по коду, артикулу, штрихкоду;

·  сканером штрихкода или из ТСД.

 

Если установлен параметр "Разворачивать по характеристикам", то при вводе номенклатуры будет выполнятся заполнение таблицы возможными характеристиками, включая пустую характеристику.

 

Нажав на кнопку «Печать» получим печатную форму ценников.

 

Регистрация программы

Обработка для полноценной работы требует регистрации в меню О программе - Регистрация на рабочем месте. На компьютере, на котором предполагается дальнейшая работа запустите обработку и откройте форму Регистрации. В форме регистрации высветиться поле "Строка ID", пошлите его нам.

Для регистрации программа записывает пароль на жесткий диск или в реестр, при этом если есть антивирусник, то он кричит об этом и спрашивает - согласится ли с тем, что программа 1с записывает что-то в реестр? нужно согласиться. Если в реестр записать не удается, программа записывает пароль на жесткий диск в каталог программы 1С или в каталог ценников. Поэтому перед регистрацией рекомендуется настроить путь к ценникам.

История версий

 

Версия 2.15 от 23.09.2011

- Добавлен вариант работы, при котором возможно назначить конкретным группам или элементам номенклатуры свои макеты. Например, все товары мы печатаем с макетом 1, а товары из группы "Спальные мешки" с макетом 2. Для работы в новом варианте на закладке "Настройки" необходимо установить галочку "Назначать макеты ценников группам и элементам справочника номенклатуры" после чего появиться новая закладка "Макеты ценников для номенклатуры". На новой закладке для каждой группы/элемента можно назначить макет. Кроме назначения макетов отдельным группам или элементам можно назначть макет всей номенклатуре, для которой не указаны явно макеты.
Старый вариант работы так же сохранен.

- Введены клавиши быстрого вызова:
 Подбор товара - F2
 Подбор нескольких товаров - F3
 Из папки - F4
 Из документа - F5
 Поиск по коду - F6
 Поиск по штрихкоду - F7
 Поиск по артикулу - F8
 Печать ценников - F9

- Поиск кода теперь осуществляется и без ввода лидирующих нулей. Например если код товара 00128, то теперь можно при поиске по коду ввести "128", программа посмотрит возможные варианты "128", "0128", "00128" и т.д. и найдет товар.

- Добавлена настройка "Запрашивать количество товара при подборе".

- Выбранный макет сохраняется. Теперь при открытии формы макет автоматически устанавливается равным последнему выбранному в предыдущем сеансе.

- Исправлена ошибка, из-за которой если штрихкод следующего товара пуст, то он устанавливается равным штрихкоду с предыдущего товара.

- Исправлена ошибка подбора товара по штрихкоду.

- Исправлена ошибка печати основного изображения на позиции, у которых изображения нет, например тара.

 

Версия 2.14 от 22.08.2011

- Добавлен новый вариант параметров цен номенклатуры, теперь можно в макете задавать цены в виде:

- ЦенаНазваниеЦены ("Цена" + слитно название цены без пробелов);

- ЦенаКодЦены ("Цена" + слитно код цены без пробелов). Например, если есть в справочнике типов цен номенклатуры цены Оптовая, Розничная для магазина, то в макете в параметрах можно указать ЦенаОптовая или ЦенаРозничнаядлямагазина. Программа сама подставит нужный тип цены для товара. Возможна подстановка для кода: если необходимо поставить в макет ценника цену с кодом 00002, то можно в параметрах написать Цена00002. Причем в форме цену можно не выбирать. Программа сама подберет нужную цену для товара. Выложены примеры макетов.
